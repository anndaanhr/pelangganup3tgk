PENDAHULUAN
Latar Belakang
Energi listrik merupakan salah satu kebutuhan utama dalam kehidupan masyarakat modern yang terus mengalami peningkatan seiring dengan pertumbuhan penduduk dan perkembangan teknologi. Kondisi ini menuntut perusahaan penyedia listrik untuk mampu memberikan pelayanan yang optimal serta didukung oleh sistem pengelolaan data yang efektif dan efisien (Priskilla et al., 2021).
Sebagai perusahaan penyedia tenaga listrik nasional, PT PLN (Persero) memiliki tanggung jawab besar dalam mengelola data pelanggan serta penjualan energi listrik. Pada tingkat unit pelayanan seperti UP3, pengolahan data pelanggan masih banyak dilakukan menggunakan spreadsheet. Metode tersebut menimbulkan berbagai kendala, di antaranya proses pengolahan data yang memakan waktu, tingginya potensi kesalahan input, serta keterbatasan dalam melakukan analisis data secara cepat dan akurat.
Selain itu, proses monitoring terhadap data pelanggan seperti pemakaian listrik, kondisi meter, serta anomali data masih belum terintegrasi secara optimal. Hal ini menyebabkan keterlambatan dalam pengambilan keputusan serta kurang efektifnya proses evaluasi kinerja penjualan energi listrik.
Untuk mengatasi permasalahan tersebut, diperlukan suatu sistem yang mampu mengelola, memproses, dan menganalisis data pelanggan secara terpusat dan terstruktur. Dalam hal ini, pengembangan sistem berbasis web dengan dukungan backend yang kuat menjadi solusi yang relevan. Backend berperan dalam mengelola logika aplikasi, memproses data, melakukan validasi, serta menyediakan layanan melalui API yang dapat diakses oleh sistem frontend.
Dengan adanya sistem backend yang terintegrasi, proses pengolahan data seperti validasi data pelanggan, analisis pemakaian listrik, serta monitoring kinerja penjualan dapat dilakukan secara lebih efisien, akurat, dan real-time. Hal ini diharapkan dapat mendukung peningkatan kualitas pelayanan serta pengambilan keputusan yang lebih tepat di lingkungan PT PLN (Persero), khususnya pada UP3 Tanjung Karang.
Tujuan Kegiatan Kerja Praktik
Adapun tujuan dari Kerja Praktik ini adalah:
Mempelajari sistem pengolahan dan manajemen data penjualan tenaga listrik di PT PLN (Persero) UP3 Tanjung Karang.
Mengembangkan sistem backend berbasis Python yang mampu mengelola data pelanggan dalam jumlah besar secara terstruktur dan efisien.
Mengimplementasikan proses pengolahan data menggunakan metode ETL (Extract, Transform, Load) untuk mengintegrasikan data pelanggan tahun 2024 dan 2025.
Membangun layanan backend (API) yang dapat menyediakan data secara real-time untuk kebutuhan visualisasi dashboard monitoring.
Menghasilkan sistem yang mampu melakukan analisis perbandingan data pelanggan antar tahun (2024–2025) untuk mengetahui tren dan perubahan pelanggan.
Mendukung pengembangan sistem monitoring berbasis web yang dapat membantu pihak manajemen dalam pengambilan keputusan berbasis data.

Sejarah Singkat
Sejarah perkembangan ketenagalistrikan di Indonesia bermula pada akhir abad ke-19, ketika perusahaan-perusahaan Belanda yang bergerak di sektor perkebunan seperti gula dan teh mulai membangun pembangkit listrik untuk kebutuhan operasional mereka sendiri. Pada masa pendudukan Jepang tahun 1942–1945, pengelolaan listrik berada di bawah kendali pemerintah Jepang. Setelah Indonesia merdeka, tepatnya pada Agustus 1945, para pemuda Indonesia mengambil alih fasilitas kelistrikan tersebut dan menyerahkannya kepada pemerintah Republik Indonesia.
Perkembangan kelembagaan kelistrikan di Indonesia selanjutnya mengalami beberapa perubahan, yaitu:
27 Oktober 1945: Dibentuknya Jawatan Listrik dan Gas oleh Presiden Soekarno.
1961: Perubahan menjadi BPU-PLN (Badan Pimpinan Umum Perusahaan Listrik Negara).
1972: Ditetapkan sebagai Perusahaan Umum Listrik Negara (Perum PLN).
1994: Status diubah menjadi Perusahaan Perseroan (Persero) dan berlaku hingga saat ini.

Sistem Informasi
Sistem informasi merupakan sekumpulan komponen yang saling berinteraksi untuk mengumpulkan, mengolah, menyimpan, dan menyajikan informasi guna mendukung kegiatan operasional dan pengambilan keputusan dalam suatu organisasi.
Sistem Informasi Berbasis Web
Sistem informasi berbasis web adalah sistem informasi yang memanfaatkan teknologi web sebagai media penyampaian informasi, sehingga dapat diakses melalui peramban tanpa instalasi khusus di sisi pengguna.
Arsitektur Client-Server
Arsitektur client–server merupakan model sistem terdistribusi yang memisahkan peran antara client sebagai pengguna layanan dan server sebagai penyedia layanan. Arsitektur ini memungkinkan pengelolaan sistem yang lebih terstruktur dan mudah dikembangkan.
RESTful API
RESTful API adalah pendekatan dalam pengembangan layanan web yang menggunakan protokol HTTP dan metode standar seperti GET dan POST untuk pertukaran data antara client dan server dengan format JSON.
Backend System
Backend system merupakan bagian sistem yang menangani logika bisnis, pengolahan data, dan interaksi dengan basis data. Backend berperan penting dalam memastikan data diproses secara benar sebelum disajikan kepada pengguna.
Bahasa Pemrograman Python
Python adalah bahasa pemrograman tingkat tinggi yang banyak digunakan dalam pengembangan backend dan pengolahan data karena sintaksnya yang sederhana dan dukungan pustaka yang luas.
FastAPI
FastAPI adalah framework web berbasis Python yang digunakan untuk membangun RESTful API dengan performa tinggi serta dokumentasi API otomatis.
Basis Data PostgreSQL
PostgreSQL merupakan sistem manajemen basis data relasional yang digunakan untuk menyimpan data pelanggan secara terstruktur dan mendukung proses query yang kompleks.
SQLAlchemy
SQLAlchemy adalah Object Relational Mapping (ORM) yang digunakan untuk mempermudah interaksi antara aplikasi Python dengan basis data PostgreSQL.
Pandas dan OpenPyXL
Pandas digunakan untuk pengolahan dan analisis data dalam jumlah besar, sedangkan OpenPyXL digunakan untuk membaca data pelanggan dari file Excel dalam proses import data.
Visualisasi Data
Visualisasi data merupakan teknik penyajian data dalam bentuk grafik atau diagram untuk mempermudah analisis tren dan perbandingan data pelanggan.

RENCANA KEGIATAN
Deskripsi Kegiatan
Dalam pelaksanaan kerja praktik di PLN UP3 Tanjung Karang yang berlangsung pada 15 Desember 2025 hingga 6 Februari 2026, penulis terlibat dalam pengembangan proyek “Perancangan dan Implementasi Backend Sistem Monitoring dan Analisis Data Pelanggan PLN Berbasis Web”. Kegiatan ini berfokus pada pembuatan aplikasi backend (server-side) yang bertugas mengelola data pelanggan listrik PLN, khususnya data tahun 2024 dan 2025, serta menyediakan antarmuka layanan (service interface) berupa RESTful API untuk mendukung kebutuhan analisis data pelanggan.
Selama periode kerja praktik, penulis bertanggung jawab merancang arsitektur sistem, mengimplementasikan kode program, dan melakukan pengujian terhadap fungsionalitas backend tersebut. Sejumlah fitur utama dikembangkan sesuai kebutuhan PLN UP3 Tanjung Karang, antara lain: mekanisme impor data pelanggan dari file Excel, proses klasifikasi pelanggan berdasarkan statusnya (misalnya apakah pelanggan tersebut aktif – tetap berlangganan, baru – mulai berlangganan pada 2025, atau hilang – berhenti berlangganan setelah 2024), fungsi analisis perbandingan jumlah pelanggan dan konsumsi listrik antara tahun 2024 dan 2025, serta penyajian informasi tren pemakaian listrik bulanan. Melalui fitur-fitur ini, sistem yang dibangun diharapkan dapat memberikan wawasan mengenai, misalnya, berapa banyak penambahan pelanggan baru dan kehilangan pelanggan pada tahun 2025 dibanding 2024, serta bagaimana pola konsumsi listrik pelanggan dari bulan ke bulan.
Teknologi yang digunakan dalam pengembangan backend ini meliputi bahasa pemrograman Python dengan kerangka kerja FastAPI untuk membangun layanan web, serta sistem manajemen basis data PostgreSQL untuk penyimpanan data. FastAPI dipilih karena kerangka kerja ini dikenal menghasilkan performa yang sangat tinggi dan mendukung pemrograman asinkron, sehingga mampu menangani banyak permintaan secara efisien. Selain itu, FastAPI memiliki fitur dokumentasi otomatis (Swagger UI) yang memudahkan penulis dan tim dalam melakukan uji coba serta integrasi API selama proses pengembangan. Sementara itu, PostgreSQL digunakan sebagai basis data karena reputasinya yang andal dalam menjaga konsistensi dan integritas data pada skala besar. Seluruh layanan backend dirancang mengikuti prinsip arsitektur RESTful dengan format pertukaran data JSON yang bersifat lintas platform, sehingga frontend (yang menggunakan kerangka Next.js) dapat mengonsumsi data dari backend secara mudah melalui jaringan intranet maupun internet.
Selama menjalankan proyek ini, penulis ditempatkan di lingkungan kerja PLN UP3 Tanjung Karang yang memberikan kesempatan untuk berkoordinasi langsung dengan pegawai dan pihak terkait. Kegiatan penulis sehari-hari meliputi diskusi rutin dengan pembimbing lapangan mengenai rancangan sistem serta pelaporan perkembangan implementasi. Diskusi tersebut mencakup demonstrasi alur pengolahan data melalui API yang dibangun, serta evaluasi bersama atas hasil sementara yang telah dicapai. Melalui interaksi intensif ini, penulis memastikan bahwa pengembangan sistem berjalan sesuai dengan kebutuhan nyata di lapangan. Apabila ditemukan kendala atau perbedaan pemahaman, penulis dapat segera melakukan penyesuaian berkat masukan dari pihak PLN. Pada akhir masa kerja praktik, backend sistem diharapkan telah selesai diimplementasikan dan dapat terintegrasi dengan frontend, sehingga PLN UP3 Tanjung Karang memiliki alat baru untuk memonitor dan menganalisis data pelanggan secara lebih efektif dan efisien dalam jangka panjang.

Pengembangan dilakukan dengan menerapkan proses ETL (Extract, Transform, Load) untuk mengolah data dari file Excel ke dalam basis data yang terstruktur. Selain itu, penulis juga merancang basis data yang mampu menangani data dalam jumlah besar serta mengembangkan API yang berfungsi untuk menyediakan data kepada sistem frontend. Sistem backend ini dirancang agar mampu memproses data secara efisien, akurat, dan mendukung kebutuhan analisis serta visualisasi data.

Data pelanggan mentah ini menjadi basis utama untuk dianalisis dan diolah oleh sistem yang dikembangkan, serta digunakan sebagai bahan uji coba fitur-fitur backend selama proses pengembangan.

Selain itu, dokumentasi resmi dan forum komunitas daring terkait teknologi FastAPI serta PostgreSQL juga dijadikan referensi teknis untuk menyelesaikan berbagai kendala selama pengembangan. Pemanfaatan data sekunder ini bertujuan memperkaya landasan teori dan metodologi, sehingga solusi backend yang dibangun memiliki justifikasi akademis serta sejalan dengan praktik terbaik (best practices) yang telah ada.

Observasi

Melalui observasi ini, penulis dapat mengidentifikasi permasalahan di alur kerja eksisting, seperti potensi duplikasi pencatatan, kesulitan dalam membandingkan data pelanggan antar tahun, serta lamanya waktu yang dibutuhkan untuk menyusun laporan manual. Hasil pengamatan tersebut menjadi masukan berharga dalam perancangan sistem backend yang diharapkan dapat mengatasi kelemahan-kelemahan proses manual tadi. Dengan kata lain, observasi membantu penulis memastikan bahwa solusi yang dikembangkan akan relevan dan sesuai dengan kebutuhan nyata di lapangan.

Dengan melakukan studi literatur, penulis memastikan bahwa pengembangan sistem didasarkan pada prinsip-prinsip ilmiah dan mengikuti perkembangan teknologi terkini, sehingga metode penyelesaian masalah yang diterapkan lebih kredibel dan teruji.
Metode Penyelesaian Masalah
Tahap awal yang dilakukan dalam kegiatan ini adalah analisis kebutuhan serta identifikasi permasalahan. Pada tahap ini, penulis menelaah kondisi awal di PLN UP3 Tanjung Karang, yaitu belum tersedianya sistem yang terintegrasi untuk memonitor data pelanggan secara real-time. Proses pelaporan dan analisis yang masih dilakukan secara manual menggunakan Microsoft Excel menyebabkan pekerjaan menjadi kurang efisien, memakan waktu yang cukup lama, serta memiliki risiko kesalahan manusia (human error) yang tinggi. Permasalahan tersebut kemudian dikonfirmasi melalui diskusi bersama pembimbing lapangan, sehingga diperoleh pemahaman yang lebih jelas mengenai kebutuhan sistem yang akan dikembangkan. Berdasarkan hasil analisis, dirumuskan kebutuhan fungsional sistem yang meliputi kemampuan untuk mengimpor data pelanggan dari file Excel ke dalam database secara akurat, kemampuan untuk mengidentifikasi perubahan status pelanggan antar tahun, serta kemampuan untuk menyajikan hasil analisis dalam bentuk informasi yang mudah dipahami oleh pengguna. Selain itu, aspek keamanan dan kerahasiaan data juga menjadi perhatian utama mengingat data pelanggan bersifat sensitif. Hasil dari tahap ini berupa spesifikasi awal sistem backend, termasuk daftar fitur yang akan dikembangkan, rancangan endpoint API, serta desain awal skema basis data.
Selanjutnya, tahap perancangan dan implementasi sistem backend dilakukan dengan merancang arsitektur sistem secara menyeluruh, yang mencakup desain basis data serta struktur API. Basis data dirancang menggunakan PostgreSQL dengan pendekatan relasional untuk menyimpan data pelanggan beserta informasi pemakaian listrik secara terstruktur. Skema database disusun sedemikian rupa agar dapat mengakomodasi data dari tahun 2024 dan 2025, sehingga memudahkan proses analisis perbandingan antar tahun. Selain itu, dirancang pula berbagai endpoint API yang mendukung kebutuhan sistem, seperti endpoint untuk mengunggah data pelanggan melalui file Excel, endpoint untuk mengambil informasi status pelanggan, serta endpoint untuk memperoleh ringkasan tren konsumsi listrik bulanan.
Implementasi sistem backend kemudian dilakukan menggunakan bahasa pemrograman Python dengan framework FastAPI yang dipilih karena memiliki performa tinggi serta mendukung pengolahan request secara efisien. Dalam proses pengembangan, digunakan berbagai pustaka pendukung seperti Pydantic untuk validasi data dan SQLAlchemy sebagai ORM untuk mempermudah interaksi dengan database PostgreSQL. Pengembangan fitur dilakukan secara bertahap, dimulai dari fitur impor data Excel dengan memanfaatkan library seperti pandas atau openpyxl untuk membaca dan memproses data. Pada tahap ini juga diterapkan proses pembersihan data (data cleaning) guna memastikan data yang disimpan memiliki kualitas yang baik. Selanjutnya, dikembangkan logika untuk mengklasifikasikan pelanggan menjadi kategori aktif, baru, dan hilang berdasarkan perbandingan data antar tahun. Selain itu, sistem juga dikembangkan untuk melakukan analisis tren konsumsi listrik bulanan yang dapat memberikan gambaran pola penggunaan listrik pelanggan. Seluruh hasil pengolahan data kemudian disajikan dalam format JSON melalui API berbasis RESTful agar dapat dengan mudah diintegrasikan dengan frontend.
Setelah tahap implementasi, dilakukan pengujian dan validasi sistem untuk memastikan seluruh fungsi berjalan dengan baik. Pengujian mencakup proses impor data, klasifikasi pelanggan, serta pengujian setiap endpoint API secara langsung melalui sistem yang telah dibangun. Hasil pengujian kemudian dibandingkan dengan perhitungan manual guna memastikan keakuratan sistem. Selain itu, dilakukan juga uji penerimaan pengguna bersama pembimbing lapangan dan pihak PLN menggunakan data nyata. Masukan yang diperoleh dari proses tersebut digunakan sebagai dasar dalam melakukan penyempurnaan sistem.
Tahap akhir dari kegiatan ini adalah implementasi sistem ke lingkungan operasional serta proses serah terima kepada pihak PLN. Sistem backend yang telah dikembangkan di-deploy ke server yang digunakan oleh PLN UP3 Tanjung Karang dengan melakukan konfigurasi yang diperlukan, termasuk pengaturan database, server aplikasi, serta aspek keamanan sistem. Setelah sistem berjalan dengan baik, dilakukan integrasi dengan frontend berbasis Next.js untuk memastikan alur kerja sistem berjalan secara end-to-end. Penulis juga menyusun dokumentasi teknis berupa API documentation yang berisi informasi mengenai endpoint, struktur data, serta cara penggunaan sistem. Proses ditutup dengan kegiatan sosialisasi dan demonstrasi sistem kepada pihak PLN, sehingga pengguna dapat memahami cara kerja dan pemanfaatan sistem secara langsung. Dengan selesainya seluruh tahapan tersebut, sistem backend yang dikembangkan diharapkan dapat membantu meningkatkan efisiensi dalam pengelolaan dan analisis data pelanggan serta mendukung pengambilan keputusan berbasis data di PLN UP3 Tanjung Karang.
